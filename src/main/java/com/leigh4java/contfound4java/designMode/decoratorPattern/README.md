# 装饰器模式

装饰器模式（Decorator Pattern）允许向一个现有的对象添加新的功能，同时又不改变其结构。这种模式创建了一个装饰类，用来包装原有的类，并在保持类方法签名完整性的前提下，提供了额外的功能。

在Java中，装饰器模式通常通过以下方式实现：

定义一个接口或抽象类，表示被装饰的对象。

创建一个具体的类实现这个接口或抽象类，这是基本的对象。

创建一个装饰器类，也实现相同的接口或抽象类，并持有一个该接口的实例。

在装饰器类中，我们可以重写接口的方法，并在调用原有对象的方法前后添加新的行为。

下面我们通过一个简单的例子来演示装饰器模式。假设我们有一个Shape接口和一个实现了该接口的Circle类。然后我们创建两个装饰器：RedShapeDecorator和ShadowShapeDecorator，分别用于给形状添加红色边框和阴影。
