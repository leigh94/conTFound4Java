# 单例模式
- 确保一个类只有一个实例，并提供一个全局访问点

## 饿汉式[EagerSingleton]
线程安全、非懒加载、性能高、不能防止反射攻击

## 懒汉式[LazySingleton]
线程不安全、懒加载、性能一般、不能防止反射攻击

## 线程安全-懒汉式[ThreadSafeSingleton]
线程安全、懒加载、性能低、不能防止反射攻击

## *双重检查锁[DoubleCheckedLockingSingleton]
线程安全、懒加载、性能高、不能防止反射攻击

## *静态内部类[StaticInnerClassSingleton]
线程安全、懒加载、性能高、不能防止反射攻击

## **枚举方式[EnumSingleton]
线程安全、非懒加载、性能高、能防止反射攻击

## 测试[SingletonDemo]
- 推荐使用：枚举方式（最简单、最安全）或静态内部类方式
- 性能考虑：双重检查锁或静态内部类
- 简单场景：饿汉式
- 避免使用：非线程安全的懒汉式
- 枚举方式是实现单例模式的最佳实践，它天生线程安全，且能防止反射和序列化破坏单例。
