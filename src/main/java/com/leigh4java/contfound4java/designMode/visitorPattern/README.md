# 访问者模式
我们使用一个计算机部件的例子来演示访问者模式。假设我们有计算机部件（如CPU、Memory、Keyboard等），我们想要为这些部件添加不同的操作（如显示信息、计算价格等），而不想修改这些部件的类。访问者模式允许我们定义一个新的操作而不改变被操作的类的结构。

步骤：

定义一个接口（ComputerPart）来表示计算机部件，它有一个接受访问者的方法。

创建具体的部件类（如Keyboard, Mouse, Monitor, Computer），实现ComputerPart接口。

定义一个访问者接口（ComputerPartVisitor），它为每个类型的部件声明一个访问方法。

创建具体的访问者类，实现ComputerPartVisitor接口，定义每个操作。

在这个例子中，我们将创建两个访问者：一个用于显示部件信息，另一个用于计算总价。

注意：访问者模式允许我们添加新的操作，而不需要修改现有的类结构，但是如果我们添加新的部件，就需要修改访问者接口和所有现有的访问者类。


访问者模式允许你在不修改现有类结构的情况下定义新的操作。它将算法与对象结构分离，让你可以定义作用于对象结构中各元素的新操作。


模式优势
开闭原则：可以引入新的访问者而无需修改现有代码

单一职责原则：将相关行为集中在一个访问者类中

灵活性：可以在运行时选择不同的访问者

解耦：将算法与对象结构分离

适用场景
需要对复杂对象结构中的元素执行多种不相关的操作

对象结构很少变化，但经常需要在此结构上定义新的操作

需要避免"污染"元素类的操作代码

访问者模式在编译器、文档处理、UI组件处理等场景中非常有用。
