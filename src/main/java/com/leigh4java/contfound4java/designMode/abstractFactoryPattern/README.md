# 抽象工厂
- 提供一个接口，用于创建相关或依赖对象的家族，而不需要明确指定具体类。

- 结构
  1. 抽象工厂（AbstractFactory）：声明一组创建抽象产品的方法。 
  2. 具体工厂（ConcreteFactory）：实现抽象工厂的方法，创建具体产品。 
  3. 抽象产品（AbstractProduct）：为每种产品声明接口。 
  4. 具体产品（ConcreteProduct）：实现抽象产品接口的具体类。

- 特点 
  1. 抽象工厂模式针对的是多个产品等级结构（产品族）。 
  2. 它强调一系列相关产品对象的创建，这些产品是相互关联或依赖的。

- 适用场景 
  1. 系统需要独立于其产品的创建、组合和表示。 
  2. 系统需要配置多个产品族中的一个。
  3. 需要强调一系列相关产品对象的设计以便进行联合使用。 
  4. 提供一个产品类库，只暴露接口而不暴露实现。
