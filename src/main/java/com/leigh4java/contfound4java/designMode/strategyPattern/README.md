# 策略模式

策略模式定义了一系列的算法，并将每一个算法封装起来，使它们可以相互替换。策略模式让算法独立于使用它的客户而变化。

在这个例子中，我们将创建一个支付策略接口，然后实现不同的支付方式。最后，我们创建一个上下文类（购物车）来使用支付策略。

步骤：

定义策略接口（PaymentStrategy）

实现具体的支付策略（AlipayStrategy, WechatPayStrategy, CreditCardStrategy）

创建上下文类（ShoppingCart）来使用策略

客户端代码演示如何使用


策略模式的优势
开闭原则：新增支付方式无需修改现有代码

消除条件语句：避免大量的if-else或switch-case

算法复用：不同的上下文可以复用相同的策略

灵活切换：运行时可以动态切换策略

职责分离：将算法实现与使用分离

实际应用场景
支付系统（如本示例）

排序算法（快速排序、归并排序、冒泡排序等）

压缩算法（ZIP、RAR、7Z等）

数据验证策略

折扣计算策略

导航算法（最短路径、最快路径等）

这个示例完整展示了策略模式在实际支付系统中的应用，体现了策略模式的灵活性和可扩展性
