# 原型模式
- 是用于创建重复对象，
- 同时又能保证性能的一种创建型设计模式。
- 它通过复制现有的实例来创建新的实例，而不是通过新建类的方式。


1. 实现原型模式通常需要以下步骤：
    - 实现Cloneable接口，该接口是一个标记接口，表示该类的对象可以被克隆。
    - 重写Object类的clone()方法，并将其访问修饰符改为public，以便外部调用。
    - 在clone()方法中，通常调用super.clone()来实现浅拷贝，如果需要深拷贝，则需对引用类型的字段进行额外的克隆。

2. 浅拷贝
   - 浅拷贝会复制对象的基本数据类型字段和引用类型字段的地址
   - 因此原始对象和克隆对象会共享引用类型字段所引用的对象

3. 深拷贝
   - 深拷贝会复制对象的所有字段，包括引用类型字段所引用的对象。
   - 因此，原始对象和克隆对象不会共享任何引用类型字段所引用的对象。

4. 关键点
   - 浅拷贝：只复制基本数据类型和引用，不复制引用指向的对象 
   - 深拷贝：复制基本数据类型，同时复制引用指向的对象 
   - Cloneable接口：Java提供的标记接口，表示对象可以被克隆 
   - clone()方法：需要重写Object类的clone()方法

5. 使用场景
   - 当创建对象的成本较高时（如需要复杂的数据库操作） 
   - 当系统需要独立于其产品的创建、构成和表示时 
   - 当需要避免使用与产品层次平行的工厂类层次时 
   - 当一个类的实例只有几个不同的状态组合时
