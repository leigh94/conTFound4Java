# 中介者模式
中介者模式用一个中介对象来封装一系列的对象交互，使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。


中介者模式总结
核心组件：

Mediator（中介者）：定义各个同事对象交互的接口

ConcreteMediator（具体中介者）：实现中介者接口，协调各个同事对象

Colleague（同事类）：每个同事类都知道它的中介者对象

优点：

减少了对象间的耦合

将对象间的一对多关联转变为一对一的关联

简化了对象协议

抽象了对象间的协作

缺点：

中介者可能变得复杂

中介者集中了控制逻辑，可能成为系统瓶颈

适用场景：

一组对象以定义良好但是复杂的方式进行通信

想定制一个分布在多个类中的行为，而又不想生成太多的子类

对象间的引用导致依赖关系混乱且难以理解

实际应用：

GUI框架中的事件处理

聊天应用

空中交通管制系统

工作流引擎

消息中间件

微服务架构中的API网关

模式变体：

简单中介者：处理基本的对象协调

复杂中介者：包含业务逻辑和状态管理

分层中介者：支持多级协调

中介者模式通过引入一个中介对象来封装一系列对象之间的交互，使对象之间不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。这种模式特别适用于对象间交互复杂且频繁变化的系统。
