# 外观模式
外观模式（Facade Pattern）为子系统中的一组接口提供一个统一的入口。它定义了一个高层接口，这个接口使得这一子系统更加容易使用。

下面是一个简单的Java外观模式demo，模拟一个计算机启动过程。计算机启动需要经过CPU、内存、硬盘等子系统的启动，但是用户只需要按一下开机按钮即可，不需要了解内部细节。
步骤：
定义子系统类：CPU、Memory、HardDrive。
定义外观类：Computer，它封装了这些子系统的复杂操作。
客户端通过外观类来简化操作。



模式分析
外观模式优点：

简化了客户端与子系统之间的交互

降低了客户端与子系统的耦合度

提供了一个统一的入口点，便于使用

外观模式适用场景：

当需要为复杂子系统提供简单接口时

当客户端与多个子系统之间存在大量依赖关系时

当需要分层构建子系统时，可以使用外观模式定义每层的入口点

在这个例子中，ComputerFacade 类隐藏了 CPU、内存和硬盘之间的复杂交互，为客户端提供了简单直观的接口（startComputer、shutdownComputer、executeTask）。
