# 迭代器模式

迭代器模式用于顺序访问集合对象的元素，而不需要知道集合对象的底层表示。

我们将创建一个包含元素的容器（例如，一个自定义的集合），并提供一个迭代器来遍历该容器。我们还将展示如何使用Java内置的迭代器。

步骤：

定义迭代器接口（Iterator），包含hasNext()和next()方法。

定义容器接口（Container），包含获取迭代器的方法。

实现一个具体的容器类（例如，一个自定义的数组集合），并实现Container接口。

实现一个具体的迭代器类，用于遍历该容器。

另外，我们也可以使用Java内置的java.util.Iterator和java.lang.Iterable来展示如何实现迭代器。

迭代器模式总结
核心组件：

Iterator（迭代器）：定义访问和遍历元素的接口

ConcreteIterator（具体迭代器）：实现迭代器接口，跟踪遍历位置

Aggregate（聚合）：定义创建相应迭代器对象的接口

ConcreteAggregate（具体聚合）：实现创建相应迭代器的接口

优点：

支持不同的方式遍历一个聚合对象

简化了聚合类的接口

在同一个聚合上可以有多个遍历

封装集合的内部实现

缺点：

增加了系统的复杂性

对于简单的集合，使用迭代器可能过度设计

适用场景：

访问一个聚合对象的内容而无需暴露它的内部表示

支持对聚合对象的多种遍历

为遍历不同的聚合结构提供一个统一的接口

实际应用：

Java集合框架（Collection Framework）

数据库结果集遍历

文件系统遍历

XML文档遍历

组合模式中的层次结构遍历

迭代器类型：

内部迭代器：迭代逻辑由迭代器自身控制

外部迭代器：客户端控制迭代过程

正向迭代器、反向迭代器

过滤迭代器、分页迭代器

复合迭代器

迭代器模式是Java中最常用的设计模式之一，它提供了一种统一的方式来访问集合元素，而不需要了解底层的数据结构
