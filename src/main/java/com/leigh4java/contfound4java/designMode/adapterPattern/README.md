# 适配器模式
- 结构型设计模式，
- 它允许接口不兼容的类能够一起工作。
- 适配器模式通常涉及一个适配器类，
- 该类将一个类的接口转换成客户端所期望的另一个接口

### 适配器模式包含以下角色：
1. 目标接口(Target): MediaPlayer - 客户端期望的接口
2. 适配者(Adaptee): AdvancedMediaPlayer - 需要被适配的现有接口
3. 适配器(Adapter): MediaAdapter - 将适配者接口转换成目标接口

### 适用场景：
- 需要使用现有类，但其接口不符合需求
- 想要创建一个可复用的类，与不相关的类协同工作
- 需要使用多个现有的子类，但为每个子类适配接口不现实

### 优点：
- 可以让任何两个没有关联的类一起运行
- 提高了类的复用性
- 增加了类的透明度
- 灵活性好
