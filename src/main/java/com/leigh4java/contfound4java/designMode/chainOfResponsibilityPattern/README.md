# 责任链模式
为请求创建了一个接收者对象的链。这种模式给予请求的类型，对请求的发送者和接收者进行解耦。在这种模式中，通常每个接收者都包含对另一个接收者的引用。如果一个对象不能处理该请求，那么它会把相同的请求传给下一个接收者，依此类推。

我们将通过一个简单的例子来演示责任链模式：假设我们有一个审批系统，不同的审批人根据审批金额有不同的权限。例如，经理可以审批1000元以下的请求，总监可以审批5000元以下的请求，总经理可以审批10000元以下的请求，超过10000元的需要开会讨论。

步骤：

创建一个抽象审批者类，其中包含指向下一个审批者的链接和审批方法。


责任链模式将请求的发送者和接收者解耦，让多个对象都有机会处理请求，将这些对象连成一条链，并沿着这条链传递请求，直到有对象处理它为止。


责任链模式总结
核心组件：

Handler（处理器）：定义处理请求的接口

ConcreteHandler（具体处理器）：具体处理请求的类

Client（客户端）：创建请求并提交给责任链

优点：

降低耦合度：请求发送者不需要知道哪个对象处理请求

增强灵活性：可以动态调整责任链

符合开闭原则：可以新增处理器而不修改现有代码

缺点：

请求可能未被处理：如果链中没有合适的处理器

性能问题：链过长可能影响性能

调试困难：请求的传递路径不明确

适用场景：

多个对象可以处理同一请求，但具体由哪个对象处理在运行时确定

需要在不明确指定接收者的情况下向多个对象中的一个提交请求

可动态指定一组对象处理请求

实际应用：

Java Servlet中的Filter链

Spring Security的认证过滤器链

审批流程系统

异常处理机制

事件处理系统
