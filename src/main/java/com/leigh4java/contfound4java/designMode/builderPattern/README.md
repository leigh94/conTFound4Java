# 建造者模式
- 允许你使用相同的创建代码生成不同类型和形式的对象。

- 模式结构
  - 产品：要创建的复杂对象 
  - 抽象建造者：定义创建产品各个部分的抽象接口 
  - 具体建造者：实现抽象建造者接口，构造和装配各个部件 
  - 指挥者：构建使用建造者接口的对象 
  - 客户端：使用指挥者和建造者创建对象


## 优点
1. 分离构造和表示：将复杂对象的构建与其表示分离 
2. 精细控制构建过程：可以控制构建的每一步 
3. 复用构建过程：相同的构建过程可以创建不同的表示 
4. 提高可读性：链式调用使代码更易读 
5. 保证不可变性：产品对象一旦创建就不可变

## 适用场景
1. 创建复杂对象，其构造过程需要多个步骤 
2. 对象有大量可选参数 
3. 需要创建不同表现形式的对象 
4. 希望创建不可变对象
